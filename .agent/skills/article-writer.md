# Article Writer Skill

Use this when the user asks to turn the project into a blog post.

Suggested structure:
1. Start with the hook: the magic is mostly a loop, tools, and state.
2. Explain the harness: everything around the model that makes it useful.
3. Walk through the code in layers: model call, tools, execution, feedback loop.
4. Show why bash is powerful and scary.
5. Introduce lazy-loaded skills as context engineering.
6. End with production gaps: sandboxing, permissions, memory, telemetry, evals.
